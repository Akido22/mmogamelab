Общая информация о генераторе оформлений

Генераторы оформлений предназначены для простого создания пользователями новых вариантов оформления для страниц своих игр. Для администратора игры генерация нового варианта оформления заключается в:

  • выборе шаблона из списка доступных;
  • загрузка крупного изображения по теме игры (если в шаблоне предусмотрено для него место);
  • ввод текста с названием игры, который необходимо разместить на шаблоне (если в шаблоне этот текст предусмотрен);
  • настройка параметров оформления (для разных шаблонов они могут быть разными): цвета фона, текста, дополнительных элементов.

Получив указанные данные, генератор сформирует эксклюзивный вариант оформления специально под данного администратора. Далее администратор может либо установить сгенерированный вариант, либо скачать себе его на компьютер, чтобы вручную произвести любые желаемые доработки. Данное руководство представляет собой техническое задание для дизайнеров, которые будут разрабатывать шаблоны. Пользователям шаблонов читать этот текст не требуется.

Чтобы разработать качественный шаблон, который программное обеспечение конструктора игр сможет превратить в оригинальное оформление для игры, необходимо придерживаться определённых правил:

  • в шаблоне должно быть предусмотрено место для крупного пользовательского изображения;
  • необходимо помнить, что изображение может оказаться как светлым, так и тёмным. Элементы, размещаемые поверх изображения, должны хорошо смотреться на любом фоне;
  • помните, что ваш дизайн, в конечном итоге, будет собираться программой. Под каждый шаблон будет требоваться работа программиста, и чем он сложнее, тем более трудоёмкой будет работа. Не применяйте к пользовательскому изображению сложных фильтров;
  • если вы делаете "резиновый" дизайн, который будет тянуться вслед за окном браузера, убедитесь, что в дизайне предусмотрены тянущиеся места. Помните, что присланное пользователем изображение не бесконечно во все стороны. Если сомневаетесь, сделайте оформление фиксированной ширины (как в примере выше);
  • набор блоков на странице не фиксирован. Если у игры нет рейтингов, то блок рейтингов показываться не будет. Ваш шаблон должен хорошо смотреться как со включенными, так и с выключенными отдельными блоками;
  • на титульной странице должны быть предумотрены следующие блоки:
    • форма входа в игру;
    • блок новостей (может отсутствовать). У новостей есть поля: дата публикации, заголовок, текст анонса и опциональная ссылка "читать далее";
    • блок рейтингов (может отсутствовать). Рейтинг может быть один, либо их может быть несколько. У каждого рейтинга есть его название ("Самые богатые"), а также табличная часть с двумя колонками — именем персонажа и значением показателя;
    • блок ссылок (может отсутствовать). Каждая ссылка — это просто текстовый элемент, например "Библиотека", "Форум". В списке ссылок часть может быть отключена. Например кнопка "Безопасный вход" при нажатии может превратиться в некликабельный текст "Безопасный вход выполнен";
    • блок счётчиков посещаемости;
    • блок для копирайта (2-3 строки);
  • вёрстка должна одинаково смотреться в популярных браузерах (IE7, IE8, Firefox, Opera, Chrome);
  • документ должен быть свёрстан в валидный XHTML и успешно проходить проверку через валидатор W3C.
