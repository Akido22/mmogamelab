Страница с выбором статического образа генерируется из двух файлов: charimage-select.html и charimage-select-layout.html. В них можно использовать следующие переменные шаблонизатора:

  • images — список образов. Каждый элемент списка — запись с полями:
    • id — идентификатор образа;
    • name — название;
    • description — описание;
    • image — ссылка на изображение;
    • price — HTML-представление цены за установку этого образа;
  • image — id образа, который должен быть выбран изначально;
  • width — ширина образа в пикселях;
  • height — высота образа в пикселях;
  • error — текст сообщения об ошибке (может отсутствовать);
  • layout — присутствует только в шаблоне charimage-select.html. В это место подставляется содержимое шаблона charimage-select-layout.html.

Чтобы программное обеспечение конструктора могло правильно отображать ход выбора образа, необходимо создать на странице элементы со следующими id:

  • charimgsel-input — hidden-поле, в которое будет записываться id выбранного образа;
  • charimgsel-image — картинка-превью, у которой будет изменяться src для отображения выбранного образа;
  • charimgsel-name — название текущего выбранного образа;
  • charimgsel-description — описание текущего выбранного образа;
  • charimgsel-error — сообщение об ошибке;
  • charimgsel-price — цена установки данного образа.

Страница для создания послойного образа генерируется из двух файлов: charimage-construct.html и charimage-construct-layout.html. В них можно использовать следующие переменные шаблонизатора:

  • layers — список слоёв. Каждый элемент списка — запись с полями:
    • id — идентификатор слоя;
    • name — название слоя;
    • images — список изображений для данного слоя. Каждый элемент списка — запись с полями:
      • id — идентификатор изображения;
      • image — ссылка на изображение;
    • image — id изображения, которое должно быть выбрано изначально;
  • width — ширина образа в пикселях;
  • height — высота образа в пикселях;
  • price — цена установки образа;
  • error — текст сообщения об ошибке для вставки в charimgsel-error (может отсутствовать);
  • layout — присутствует только в шаблоне charimage-construct.html. В это место подставляется содержимое шаблона charimage-construct-layout.html.

Чтобы программное обеспечение конструктора могло правильно отображать ход создания образа, необходимо создать на странице элементы со следующими id:

  • charimgsel-layer-"кодслоя" — это div, которому будет установлено свойство background-image в выбранное изображение слоя. Стек вложенных друг в друга divs позволяет создать превью для конструируемого образа;
  • charimgsel-input-"кодслоя" — hidden-поле формы, в которое будет помещен id выбранного изображения для слоя;
  • charimgsel-error — сообщение об ошибке.
