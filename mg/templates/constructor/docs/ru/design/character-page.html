Внутриигровая страница персонажа является внутренней страницей игрового интерфейса. Шаблон задаётся в файле character-page.html игрового оформления. В нём можно использовать переменные шаблонизатора:

  • char — ссылка на объект Character скриптового движка;
  • character.name — имя персонажа;
  • character.html — HTML-представление имени персонажа;
  • character.sex — пол персонажа (0 — мужской, 1 — женский);
  • character.avatar — HTML-представление аватара персонажа;
  • character.params — список всех параметров персонажа (может отсутствовать). Каждый параметр — это запись. Если у неё установлено поле header, то это заголовок. header содержит текст заголовка. Иначе — это информация о параметре, и запись имеет поля name (название) и value (значение);
  • character.actions — список ссылок. Каждый элемент списка — это запись с полями:
    • href — ссылка;
    • text — текст ссылки.

Аватар персонажа, который помещается на страницу информации, может содержать образ персонажа и, например, слоты экипировки вокруг него. Шаблон задаётся в файле character-page-avatar.html игрового оформления. В нём можно использовать переменные шаблонизатора:

  • equip_layout — если этот параметр присутствует, значит страница верстается при помощи модуля экипировки. В этом случае он имеет следующие поля:
    • items — элементы разметки. Каждый элемент определяет объект, который должен быть размещён на поле. У него есть поля:
      • cls — CSS-класс;
      • width — ширина в пикселях (без учёта рамки);
      • height — высота в пикселях (без учёта рамки);
      • x — смещение по X от левого края блока с разметкой до рамки объекта;
      • y — смещение по Y от верхнего края блока с разметкой до рамки объекта;
      • border — толщина рамки вокруг объекта;
      • image — изображение для объекта (может отсутствовать). Если присутствует, то имеет поля:
        • src — ссылка на картинку;
        • alt — альтернативный текст для картинки, пока она не прогрузилась;
        • title — атрибут title для картинки;
        • onclick — JavaScript-команда при клике на объект (может отсутствовать);
      • hint — подсказка при наведении мыши (может отсутствовать). Если присутствует, то имеет поля:
        • cls — CSS-класс объектов, над которыми должна всплывать подсказка;
        • html — текст подсказки (с экранированными символами по правилам JavaScript);
    • avatar_image — ссылка на картинку с образом персонажа (используется, если не задан equip_layout);
