За оформление списка разделов форума отвечает файл index.html в шаблоне оформления социальной части игры. Он может использовать следующие переменные шаблонизатора:

  • categories — список разделов (а также всяких подзаголовков между ними). У каждой записи есть поля:
    • category — если этот признак установлен, значит искомая запись — собственно раздел форума. Сам category является структурой со следующими полями:
      • id — идентификатор раздела;
      • title — название раздела;
      • description — описание раздела;
      • topics — количество топиков в разделе (может отсутствовать);
      • replies — количество ответов в разделе (может отсутствовать);
      • unread — признак, что в разделе есть непрочитанные темы;
      • lastinfo — информация о последнем сообщении в разделе (может отсутствовать). Если присутствует, то это структура с полями:
        • topic — идентификатор топика;
        • post — идентификатор поста;
        • subject_html — тема;
        • updated — дата обновления;
        • author_html — автор поста;
    • header — если этот признак установлен, то искомая запись — заголовок. Сам header является текстом заголовка;
    • delim — признак того, что необходимо нарисовать разделитель.

За оформление списка топиков в разделе отвечает файл category.html в шаблоне оформления социальной части игры. Он может использовать следующие переменные шаблонизатора:

  • rules — правила, которые необходимо отобразить вверху (может отсутствовать);
  • topics — список топиков (может отсутствовать). У каждого топика есть поля:
    • uuid — идентификатор топика;
    • pinned — тема закреплена;
    • closed — тема закрыта;
    • unread — признак, что топик не прочитан;
    • subject_html — тема;
    • subscribed — признак, что пользователь подписан на тему;
    • literal_created — дата создания (написанная словами);
    • pages — если присутствует, то необходимо показать блок навигации по страницам. Pages — это список, каждый элемент которого имеет следующие поля:
      • entry.a — если элемент является ссылкой, то entry.a.href будет содержать URL. Если entry.a пустой, значит это не ссылка;
      • entry.text — текст элемента;
    • author_html — автор топика;
    • posts — количество ответов в топике (может отсутствовать);
    • last_post — последний ответ (может отсутствовать);
    • last_post_page — страница, на которой находится последний ответ (имеет смысл, только если установлен признак last_post);
    • last_post_created — время последнего ответа (имеет смысл, только если установлен признак last_post);
    • last_post_author_html — автор последнего ответа (имеет смысл, только если установлен признак last_post);
  • message — сообщение в низу страницы (может отсутствовать);
  • rules_above — сообщения от модуля "Интерактивные правила форума" (если надо показать правила над списком тем). Каждое правило — это структура:
    • text — текст правила.

За оформление самого топика отвечает файл topic.html в шаблоне оформления социальной части игры. Он может использовать следующие переменные шаблонизатора:

  • topic — структура, описывающая топик:
    • uuid — идентификатор топика;
    • subject_html — тема топика;
    • subscribed — признак того, что пользователь подписан на топик;
    • literal_created — дата создания (написанная словами);
    • avatar — аватар автора топика;
    • author_html — имя автора топика;
    • status — статус автора топика (может отсутствовать);
    • author_menu — меню ссылок рядом с аватаром автора (может отсутствовать). Если присутствует, то является списком следующих структур:
      • href — ссылка;
      • title — текст ссылки;
    • content_html — текст топика;
    • tags_html — теги (могут отсутствовать);
    • signature — подпись автора топика (может отсутствовать);
    • topic_actions — доступные действия над топиком (могут отсутствовать);
  • show_topic — признак того, что необходимо показать содержимое самого поста (бывает, что сам пост показывать не надо, а надо показывать только ответы);
  • posts — список постов (ответов). Каждый пост — это структура со следующими полями:
    • uuid — идентификатор поста;
    • literal_created — дата поста (написанная словами);
    • avatar — аватар автора поста;
    • author_html — имя автора поста;
    • status — статус автора поста (может отсутствовать);
    • author_menu — меню ссылок рядом с аватаром автора (может отсутствовать). Если присутствует, то является списком следующих структур:
      • href — ссылка;
      • title — текст ссылки;
    • title_html — заголовок поста (может отсутствовать). Присутствует только в результатах поиска;
    • content_html — текст поста;
    • tags_html — теги (могут отсутствовать);
    • signature — подпись автора поста (может отсутствовать);
    • post_actions — доступные действия над постом (могут отсутствовать);
  • pages — если присутствует, то необходимо показать блок навигации по страницам. Pages — это список, каждый элемент которого имеет следующие поля:
    • entry.a — если элемент является ссылкой, то entry.a.href будет содержать URL. Если entry.a пустой, значит это не ссылка;
    • entry.text — текст элемента;
  • new_post_form — место, куда будет помещаться форма написания нового поста.

За оформление страницы со списком тегов отвечает файл tags.html в шаблоне оформления социальной части игры. Он может использовать следующие переменные шаблонизатора:

  • tags — список тегов. Каждый элемент имеет следующие поля:
    • url — ссылка;
    • html — сам тег.

За оформление правил, отображаемых модулем "Динамические правила", отвечает файл rules.html. Он может использовать переменные шаблонизатора:

  • title — заголовок;
  • rules — список правил. Каждое правило имеет поля:
    • text — текст правила.
