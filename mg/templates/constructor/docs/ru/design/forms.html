Оформление игровых форм

Администратор игры имеет возможность настраивать web-формы, которыми игроки пользуются во время игры. Форма — это универсальный шаблон, используемый во многих интерфейсах. Например:

  • настройки игры;
  • анкета персонажа;
  • форма создания темы на форуме;
  • форма отправки комментария;
  • формы редактирования тем и комментариев;
  • настройки форума;
  • форма ввода e-mail для напоминания пароля;
  • и т.д.

Отдельно настраиваются шаблоны формы в игровом интерфейсе и интерфейсе социалки. Формат у файлов абсолютно идентичный.

Формат шаблона form.html

  • form_method — метод формы (get/post);
  • form_action — URL, куда отправлять форму при отправке;
  • form_hidden — список элементов input type="hidden", каждый элемент содержит поля:
    • name — название поля;
    • value — его значение;
  • form_top — надписи над формой. Каждый элемент имеет поле:
    • html — html-текст, который надо разместить;
  • form_bottom — надписи под формой. Каждый элемент имеет поле:
    • html — html-текст, который надо разместить;
  • form_cols — количество колонок в полях ввода;
  • form_rows — список строк формы. В каждой строке может быть одна или несколько колонок. В каждой колонке может содержаться один элемент формы (или не содержаться ни одного). Каждая строка содержит поля:
    • fst — признак, что строка является первой;
    • lst — признак, что строка является последней;
    • width — ширина одной колонки в данной строке в процентах. Например, если в строке 4 колонки, то в поле width будет записано 25;
    • show_header — надо ли показывать заголовок у данной строки формы;
    • cols — список колонок. Каждая колонка, в свою очередь, имеет свои поля:
      • name — название поля (код, который будет передан на сервер);
      • value — значение поля (текст поля ввода, код выбранного элемента в селекторе и т.д., html-encoded);
      • desc — описание поля;
      • error — показать ошибку (может отсутствовать). Содержит поля:
        • text — текст ошибки;
      • td_class — CSS-класс ячейки таблицы (может отсутствовать);
    • padding_top — признак, что строке нужен дополнительный отступ сверху;
    • element_header — признак, что этот элемент формы является заголовком. Содержит поле:
      • html — html-содержимое заголовка;
    • element_input — признак, что этот элемент является строкой ввода;
    • element_quantity — поле ввода количества (с кнопками "плюс" и "минус");
    • min_value — минимальное значение для поля ввода количества;
    • max_value — максимальное значение для поля ввода количества;
    • element_range — признак, что этот элемент является полем ввода диапазона (два числа — "от" и "до");
    • value_min — минимальное значение для поля ввода диапазона;
    • value_max — максимальное значение для поля ввода диапазона;
    • element_checkbox — признак, что этот элемент является чекбоксом ("галочкой");
    • onclick — javascript-код, который должен сработать при клике на чекбокс;
    • element_radio — признак, что этот элемент является чекбоксом ("галочкой");
    • form_control_id — уникальный идентификатор элемента (используется в радиокнопке);
    • checked — признак, что чекбокс или радиокнопка отмечены;
    • text — текст, который надо вывести рядом с чекбоксом или радиокнопкой;
    • element_file — признак, что этот элемент является кнопкой выбора файла для загрузки;
    • element_password — признак, что этот элемент является строкой ввода пароля;
    • element_select — признак, что этот элемент является выпадающим списком выбора;
    • options — допустимые варианты выбора для выпадающего списка. Каждый элемент имеет поля:
      • value — код значения, который будет отправлен на сервер;
      • selected — признак, что данный элемент выбран;
      • bgcolor — цвет фона элемента (может отсутствовать);
      • text — текст, который необходимо показать пользователю;
    • element_textarea — признак, что данный элемент является многострочным полем ввода;
    • form_textarea_cols — количество колонок текста в поле типа textarea или texteditor;
    • form_textarea_rows — количество строк текста в поле типа textarea или texteditor;
    • element_textarea_fixed — признак, что данный элемент является многострочным полем ввода с фиксированной шириной;
    • element_texteditor — признак, что данный элемент является многострочным полем ввода, допускающим использование bb-code (отображается соответствующая панель кнопок);
    • fixed_not_ok — используется в texteditor. Если признак установлен, значит необходимо растянуть поле на всю ширину;
    • attaches — отобразить кнопки вставки изображений;
    • show_smiles — признак требует отобразить панель со смайликами для этого поля;
    • element_submit — признак, что данный элемент является кнопкой отправки формы;
    • submit_name — если это поле установлено, то оно указывает значение атрибута name у кнопки (используется, чтобы различать на стороне сервера нажатые кнопки формы, если их несколько);
  • smile_categories — если присутствует, то определяет каталог доступных смайликов. Каждый элемент списка — это описание раздела с полями:
    • id — идентификатор раздела;
    • name — название раздела;
    • smiles — список смайликов в этом разделе. У каждого смайлика есть поля:
      • code — код смайлика;
      • image — URL картинки;
  • errors_on_top — признак, нужно ли дополнительно показывать все ошибки в заполнении формы наверху;
  • form_texteditors — есть ли в форме хотя бы один текстовый редактор (соответственно, надо ли подключать javascript, отвечающий за них).
  • form_quantities — есть ли в форме хотя бы одно поле ввода количества (соответственно, надо ли подключать javascript, отвечающий за них).

Кроме того, в шаблоне можно использовать параметры, представляющие собой просто перевод надписей на текущий язык игры:

  • bold — полужирный;
  • italic — курсив;
  • underline — подчёркивание;
  • strike — зачёркивание;
  • quote — цитирование;
  • code — код;
  • red — красный;
  • green — зелёный;
  • blue — синий;
  • dark_green — тёмно-зелёный;
  • magenta — фиолетовый;
  • yellow — жёлтый;
  • orange — оранжевый;
  • image — изображение;
  • insert_image — вставить изображение;
  • translit — транслит;
  • transliterate_to_russian — транслитерация на русский.
