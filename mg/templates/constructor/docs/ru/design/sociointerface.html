Общая информация о HTML-шаблонах в конструкторе

Шаблон оформления для страниц форума и блогов должен удовлетворять следующим требованиям:

  • в пакете должно быть ровно один html-файл, и он должен называться index.html;
  • в пакете должен быть, как минимум, один css-файл.

При формировании страницы социального интерфейса шаблонизатор берёт файл index.html из пакета с оформлением и вместо тега [%content%] подставляет содержимое, которое необходимо отобразить. Четырьмя основными видами содержимого являются:

  • список разделов форума;
  • список тем в разделе форума;
  • тема с ответами;
  • облако тегов.

На то, как формируется внутренний контент, пользователь влиять не может. Он может изменить глобальный шаблон index.html, чтобы задать общую структуру страниц, оформить шапку, подвал, меню и т.д. Основным способом воздействия на отображение социальных страниц является редактирование css-файла. Многие элементы, присутствующие на страницах форума, имеют свои id и class, пользуясь которыми, можно изменять их внешний вид. Внизу есть ссылка на скачивание актуальных шаблонов для контента, в которых вы можете посмотреть id и class элементов оформления.

В файле index.html вам доступны следующие параметры шаблонизатора:

  • title — заголовок страницы;
  • head — дополнительные теги, которые следует поместить в раздел <head> документа;
  • htmlmeta.description — описание страницы (для поисковых систем);
  • htmlmeta.keywords — ключевые слова (для поисковых систем);
  • topmenu_left — левое верхнее меню. Каждый пункт меню имеет параметры:
    • html — текст пункта меню;
    • href — если присутствует, то является ссылкой на страницу, куда должна вести ссылка. Если отсутствует, значит текст не должен быть кликабельным;
    • header — если является заголовком (заметным выделенным текстом);
  • topmenu_right — правое верхнее меню. Каждый пункт меню имеет параметры:
    • html — текст пункта меню;
    • href — если присутствует, то является ссылкой на страницу, куда должна вести ссылка. Если отсутствует, значит текст не должен быть кликабельным;
    • image — если присутствует, то является ссылкой на иконку рядом с пунктом меню;
  • menu_left — левое меню непосредственно над контентом страницы. Каждый пункт меню имеет параметры:
    • html — текст пункта меню;
    • href — если присутствует, то является ссылкой на страницу, куда должна вести ссылка. Если отсутствует, значит текст не должен быть кликабельным;
  • menu_right — правое меню непосредственно над контентом страницы. Каждый пункт меню имеет параметры:
    • html — текст пункта меню;
    • href — если присутствует, то является ссылкой на страницу, куда должна вести ссылка. Если отсутствует, значит текст не должен быть кликабельным;
